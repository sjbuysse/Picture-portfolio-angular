Loader

1. Implementation

1.1. Adding the loader component

To add a loader to the application, import the LoaderComponent from the Arizona Pattern Library into the application's root module app.module.ts.

import {{ '{' }} LoaderComponent {{ '}' }} from 'arizona-pattern-library/lib/components/loader/loader.component';
...

@NgModule({{ '{' }}
	...
	declarations: [Loader]
{{ '}' }})
export class AppModule {{ '{' }} {{ '}' }}

1.2. Using the loader component

You can use the arz-loader tag in a components template. The tag has the following attributes:
  • busy: boolean to display or hide the loader component
  • options: (Optional) Optional parameters to customize the loader.

1.2.1. Busy

This is an a boolean representing whether the loader should be displayed or not.

1.2.1. Options

The options parameter should implement the LoaderOptions interface. 
interface LoaderOptions {{ '{' }}
	size?: string;
	borderWidth?: string;
	movingColor?: color;
	backgroundColor?: color;
{{ '}' }}
export type color = 'primary' | 'secondary' | 'tertiary' | 'success' | 'info' | 'warning' | 'danger'| 'light' | 'dark';
  • size: The size of the loader in pixels, provided as a string. Default "40".
  • borderWidth: The width of the border in pixels, provided as a string. Default "4".
  • movingColor: The color of the spinning part, provided as a string. Default "primary".
  • backgroundColor: The background color of the spinner, provided as a string. Default "light".

2. Examples

2.1 Default loader


<arz-loader [isBusy]="(isDefaultBusy$ | async)"></arz-loader>

2.2 Customized loader


<arz-loader [isBusy]="(isCustomizedBusy$ | async)" [options]="options"></arz-loader>
